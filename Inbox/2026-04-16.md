修订后的个人本地使用项目设计（纯本地、配置文件、TXT 日志）

---

### 一、目标
- 构建一个仅在本机运行的 API 中转代理，供个人调试/使用。  
- 所有敏感信息（API‑key、base‑url、模型名等）均保存在项目目录下的 **config.yaml / .env**（任选其一），不上传、不共享。  
- 使用记录（请求时间、供应商、模型、消耗的 token、返回状态）直接写入本地 **usage.txt**（追加模式），无需数据库。  
- 功能保持：  
  1. 输入各大供应商的 API‑key、base‑url → 自动拉取该供应商可用模型列表（付费/免费）。  
  2. 支持自定义供应商（需要提供 API‑key、base‑url、model）。  
  3. 适配不同供应商的 API 模式（openai‑completions、chat_completions）以及是否原生支持 OpenAI 调用方式。  
  4. 提供页面展示各供应商对应模型的 token 消耗统计（近 3h、24h、今日、本周、本月、当年），统计基于 **usage.txt** 的离线分析。

### 二、架构概览（简化版）

```
+----------------+        +----------------+        +-----------------+
|   前端 (React) | <===>  |   后端 (FastAPI) | <===>  | 本地文件/进程   |
+----------------+        +----------------+        +-----------------+
        ^                        ^                        ^
        |                        |                        |
        |   +----------------+   |   +----------------+   |
        |   | 配置加载器     |   |   | 模型发现/缓存    |   |
        |   +----------------+   |   +----------------+   |
        |                        |                        |
        |   +----------------+   |   +----------------+   |
        |   | 供应商注册表   |   |   | 转发适配器     |   |
        |   +----------------+   |   +----------------+   |
        |                        |                        |
        |   +----------------+   |   +----------------+   |
        |   | 日志写入器 (TXT) |   |   | 统计聚合器     |   |
        |   +----------------+   |   +----------------+   |
```

- **前端**：React + Ant Design（或任意轻量 UI），负责：
  - 配置页面（读取/写入 local config 文件）。
  - 供应商列表展示 & 自定义供应商添加。
  - 模型测试页（选择供应商+model，发送示例请求）。
  - 使用统计页面（读取 usage.txt，按时间窗口聚合并展示图表）。

- **后端**：FastAPI（单进程），提供：
  - `/config` 读取/写入本地配置文件（仅供前端使用，安全起见只读敏感字段的掩码）。
  - `/providers` CRUD（实际写入本地 `providers.json`）。
  - `/models/{provider}`：返回已缓存的模型列表（若无则实时拉取并缓存到 `models_cache.json`）。
  - `/v1/chat/completions` 与 `/v1/completions`：统一转发入口。
  - `/usage/stats`：读取 `usage.txt` 并返回各时间窗口的聚合结果。
  - 内部 **适配器层**：为每类供应商实现 `ProviderAdapter`（请求构造、响应解析、模型获取）。
  - **日志写入器**：每次成功转发后，向 `usage.txt` 追加一行 JSON 行（时间戳、provider、model、tokens、status_code、latency）。

- **本地持久化**（均放在项目根目录下的 `data/` 目录）：
  - `config.yaml`（或 `.env`）：用户填写的供应商密钥等。
  - `providers.json`：供应商元信息（名称、base_url、auth_header、api_mode、是否 OpenAI 兼容）。
  - `models_cache.json`：{provider: [model1, model2, …]}，带时间戳以判断是否过期（默认 1 小时）。
  - `usage.txt`：每行 JSON，易于用 `grep`、`jq` 或后端脚本离线分析。

### 三、核心模块与职责

| 模块 | 职责 | 关键实现点 |
|------|------|------------|
| **配置加载器** | 读取 `config.yaml` / `.env`；提供给后端其它模块使用（密钥仅在内部使用，不对外暴露明文）。 | 使用 `pydantic.BaseSettings` 或 `python-dotenv` 读取；返回脱敏版供前端展示。 |
| **供应商注册表** | 持久化供应商元信息（除密钥外）到 `providers.json`；提供增删改查接口。 | 文件操作采用原子写入（写临时文件→改名），防止意外中断导致文件损坏。 |
| **模型发现/缓存** | 根据供应商配置调用其模型列表接口（如 `/v1/models`），解析后写入 `models_cache.json`；若供应商不提供列表则允许手动维护。 | 缓存带时间戳，过期后重新拉取；提供手动刷新按钮。 |
| **转发适配器** | 抽象不同供应商的请求/响应差异；根据供应商的 `api_mode` 和 `is_openai_compatible` 选择对应适配器。 | 采用工厂模式：`AdapterFactory.get_adapter(provider)`；每个适配器实现 `build_request()`, `parse_response()`, `get_models()`。 |
| **日志写入器** | 每次转发成功后，把请求元数据写入 `usage.txt`（一行 JSON）。 | 使用 `a+` 模式追加；防止并发写入冲突，使用文件锁（`fcntl` 或 `portalocker`）。 |
| **统计聚合器** | 读取 `usage.txt`，按给定时间窗口（最近 3h、24h、今日、本周、本月、当年）计算每个供应商/模型的 token 总数、请求次数等。 | 先读取全部行到内存（数据量小，个人使用足够），按时间过滤后使用 `collections.Counter` 汇总；返回 JSON 供前端绘图。 |
| **前端页面** | 配置页、供应商管理页、模型测试页、使用统计页。 | 使用 Ant Design 表格、表单、图表（Ant Design Charts 或轻量级 Chart.js）完成 UI。 |

### 六、风险与对策（个人使用场景）

| 风险/问题 | 说明 | 对策 |
|----------|------|------|
| 供应商不提供模型列表接口 | 部分私有或老旧服务仅固定模型。 | 在 `providers.json` 中加 `static_models` 字段；若为空则允许用户在前端手动添入模型；缓存读取时优先使用静态列表。 |
| token 统计不准确（供应商未返回 usage） | 有些供应商只返回文本，不给 token 数。 | 在日志写入时，若响应中无 `usage` 字段，则使用简单估算（如 `len(prompt) + len(response)` 按字符数除以 4 估算 token），并在日志中标记 `estimated: true`。 |
| 配置文件泄露 | 误将 `config.yaml` 提交到 git。 | 在根目录加入 `.gitignore`，列出 `data/`、`*.env`、`*.yaml`；提交前运行 `git status` 检查。 |
| 并发写入 usage.txt 导致行损坏 | 前端可能快速发起多请求。 | 日志写入使用文件锁（`fcntl` 或 `portalocker`），确保每次追加为原子操作。 |
| 本地资源耗尽（日志文件过大） | 长时间使用会产生大量日志。 | 提供日志轮转脚本（每日大小 > 10M 自动归档为 `usage_YYYYMMDD.txt` 并开始新文件），或在前端提供“清除日志”按钮。 |
| 开发环境依赖冲突 | 前端/后端依赖版本不匹配。 | 使用 `venv` 为后端、`nvm` 或 `npm` 锁定版本；在 `README` 中明确写出推荐版本。 |
